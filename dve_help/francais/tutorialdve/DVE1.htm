Introduction à l’Editeur Malted (DVE):

1. Structure de l’Unité de Malted

Une Unité ou Projet Malted se compose de 3 composantes fondamentales:


A ) frames (.xml): ils forment le noyeau de l’unité. Il s’agit des différents écrans qui forment le frame.forman el núcleo de la unidad. Son las distintas pantallas de las que se compone.

B) cours (.cxml): c’est le fil conducteur des frames. Il contient l’ordre dans lequel apparaissent les frames et le reste des liens qui existent dans le frame de l’unité (comme les liens dans un menu qui amène à d’autres frames).

 C) fichiers média utilisés dans les frames:
           - audios (.mp3 / .wav / .mid) 
           - images (.jpg /.gif / .png)
           - textes (.txt / .rtf)
           - vidéo (.mpg / .avi / .mov)


Les Projets ou Unités se trouvent dans le dossier “projects” dans le répertoire Malted2. Chaque projet contient 5 dossiers dans lesquels on enregistre les différents fichiers: audio, image, text, video, xml.

Le dossier “xml” contient le cours (cxml) et tous les frames (xml). Le reste des dossiers contiennent les fichiers méda qui s’emploient dans chaque frame:


En ouvrant l’Editeur Malted (DVE), le premier pas consiste à créer une Unité ou Projet avec un nom nouveau. Ceci cré, d’une façon automatique, le dossier de l’unité avec le nom qu’on lui a assigné avec le reste des sous-dossiers audio, image, text, video, xml. La façon la plus simple d’introduire images, sons, textes, etc à l’intérireur de l’unité, consiste à introduire les fichiers souhaités à l’intérieur des dossiers correspondants (audio, image, texte, vidéo) avant d’ouvrir l’Editeur Malted (DVE). On peut aussi importer ces fichiers depuis l’Editeur. Ils seront ainsi disponibles pour être introduits dans le Frame.
